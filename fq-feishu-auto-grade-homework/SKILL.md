# 自动批改飞书作业

## 用途

- 拉取"上行部落"每日作业（飞书Wiki/Docx）
- 给出评分（优秀/合格/不合格）和评语
- 生成批改总结（可选写入飞书多维表格或推送到群聊）
- 提醒未提交成员

## 前置条件

1. 已有 Feishu 应用，具备以下权限：
   - `docs:doc:readonly` - 读取文档
   - `wiki:wiki:readonly` - 读取知识库
   - `drive:drive:readonly` - 读取云空间
   - `docs:document.comment:create` - 创建评论（如需写评语）
2. 目标文档已共享给应用
3. 环境变量已配置（可选）：
   - `FEISHU_APP_ID`
   - `FEISHU_APP_SECRET`

## 执行流程

### Step 1: 确定作业日期

- 默认批改"昨天"的作业
- 用户可指定日期，如"批改4月15日作业"

### Step 2: 拉取作业文档

1. 从总文档（Wiki/Docx）获取小组成员的月度作业文档链接
2. 逐一读取各成员的作业文档内容
3. 定位到指定日期的作业部分

### Step 3: 批改作业

对每份作业进行评价：

**第一题评分维度：**
- 结构清晰度（是否有层次、有逻辑）
- 分析深度（是否触及本质，而非表面罗列）
- 个人思考（是否有自己的观点或案例）
- 数据支撑（是否有具体数字或历史事件佐证）

**第二题评分维度：**
- 是否如实记录副业进展
- 有无具体动作或成果

**评分标准：**
- 优秀（85-100分）：结构清晰、分析深入、有独立思考、有数据支撑
- 良好（75-84分）：结构完整、有一定分析、有个人观点
- 合格（60-74分）：达到字数要求、有基本内容
- 不合格（<60分）：未提交或明显敷衍

### Step 4: 输出批改结果

**⚠️ 重要：敏感内容处理**

如果作业内容涉及以下敏感主题，批改结果必须使用概括性表述，**不得直接引用原文内容**：

- 地缘政治（国际冲突、领土争端、大国博弈）
- 政治敏感话题
- 其他可能触发平台审核的内容

**正确的批改方式：**
```
✅ 概括性评价：
- "从多个层面分析了问题，结构清晰"
- "有历史脉络梳理，结论落地到生活影响"
- "分析有数据支撑，逻辑层次分明"

❌ 错误方式（直接引用）：
- 直接复制原文中的敏感内容
- 大段引用涉及敏感主题的句子
```

**输出格式：**

```markdown
## 📊 X月X日作业批改总结

### 第一题批改情况

| 成员 | 得分 | 评价 |
|------|------|------|
| 张三 | 92分 | ✅ 结构清晰，分析深入，有数据支撑 |
| 李四 | 85分 | ✅ 有层次，但可以更深入 |
| 王五 | 70分 | ⚠️ 内容较浅，缺少个人思考 |

### 第二题（副业记录）情况

| 成员 | 评价 |
|------|------|
| 张三 | ✅ 有具体进展和计划 |
| 李四 | ⚠️ 简单记录 |
| 王五 | ❌ 无记录 |

### 🏆 本日优秀作业

**张三** - 92分
- 亮点：结构清晰，分析有深度
- 建议：可进一步补充具体案例

### 📌 整体建议

1. 答题要有个人痕迹，避免纯资料堆砌
2. 副业要持续记录，即使只是小进展
3. 分析要落地，能和日常生活关联

### ❌ 未提交成员

- 赵六（请补交）
```

### Step 5: （可选）写入飞书或发送通知

如需将结果持久化：

1. **写入多维表格**：使用 `feishu_bitable_create_record` 或 `feishu_bitable_update_record`
2. **发送群消息**：使用 `message` 工具发送到指定群
3. **写入文档评论**：使用 `feishu_doc` 的 `create_comment` action

## 安全注意事项

1. **敏感内容自动检测**：批改时自动识别敏感内容，使用概括性表述替代直接引用
2. **不泄露隐私**：不在公开渠道输出成员的个人信息
3. **权限最小化**：仅申请必要的飞书权限
4. **结果可控**：批改结果先在对话中展示，用户确认后再持久化

## 常见问题

**Q: 作业内容被系统拦截怎么办？**
A: 使用概括性表述，不直接引用敏感内容。如"分析有深度"、"结构清晰"等评价语。

**Q: 如何处理AI生成的作业？**
A: 在评语中标注"疑似AI生成"，评分标准降低一个档次。

**Q: 成员未提交作业怎么办？**
A: 在总结中列出未提交成员名单，可后续发送提醒。
